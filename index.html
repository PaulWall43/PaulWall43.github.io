Some past work

  • Tether

    An application that brings the modern world to the classroom.


    Tether combines the best of live-updating features and iBeacon technology in order to create a new experience for students and teachers in the classroom.

    Together with the lead mobile architect at Education First and one other co-worker (shoutout to Kenneth Dai from Jiansu Provice, China) we built this mobile prototype application in five weeks in order to demonstrate innovative ideas for the classroom.

    In an effort to make the classroom more inviting and easier for the teacher to manage, we created multiple modules that sported helpful and innovative classroom features. These modules included a native voting system where all students within a given area could participate in a poll. The live poll feature is curretly being modified into an open source module avaliable for free use.We also implemented a live chatroom handled through the Firebase database API for students to use with each other during class time to ask and answer questions.

    My personal favorite feature was the reactive quiz that analyzed competency through an associative knowledge base and adjusted the quiz to match student knowledge. Smaller features include a programmable outline, a programmable game for teachers to tailor to their student as well as an accessible database of all previous classes in the system.

    If you're interested check out my LinkedIn for more information.

  • Go

    An ancient board game to improve my skills.

    I began developing this board game during my first computer science class at UCSD in order to get some extra practice with development. In between classes, during my breaks, and over summer's I've been working on perfecting the simple, yet complex rules for this game. I ended up learning more than I bargained for about object oriented programming as well as efficient graph transversal algorithms such as DFS, BFS, Depth Limited Search, A* search, and Iterative Deepening Search.

    Just recently I've begun implementing a basic AI for this. I plan to use a combination of deep learning and mini-max evalutations to make a competetive algorithm for winning games. I also plan to make the plug in accessible online for people to play their own AI against. If you want to follow along with the project hop on over to my github and check out the repository named "Go".

  • Triton Battles

    Competition that inspires academic exellence.


    Triton Battles is a website where students can compete against themselves and each others by using user-created index cards. The competition aspect of the website motivates students to learn in a different way and thus can aid in the retention of information and critical thinking under pressure.

    Our team used a MEAN stack for development, so I learned about node.js, express.js and mongo. Triton Battles also supports a unique, programmable index card, where questions can be answered by code. The user code is then compiled and run by our application and reviewed for correctness.

  • Foodar

    We'll find the restaurant for you.


    Foodar is a the mobile application that will finally pick a restaurant for you. This app was built with the indecisive individuals or groups that can never seem to make up their mind of where to eat. Instead of choosing yourself you can just tell us what kind of restaurant you want and we'll find one for you! Or if you have some restaurants in mind but can't decide we even choose from that pool for you.

    Foodar was built in Swift and is actually still in development. My partners and I are currently working on developing new features that support machine learning and personalized restaurant accounts. Although the code is not open-source, the application will be releaesd to the Apple App Store very soon. It currently uses the Yelp API to fetch and retrieve restaurants.
